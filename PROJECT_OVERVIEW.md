# LanTalk 项目介绍与当前进度

## 项目概述

LanTalk 是一个面向局域网场景的桌面即时通讯应用，基于 Wails、Go 和 Vite 构建。项目目标是在不依赖中心服务器的前提下，让同一局域网内的多台电脑可以自动发现彼此，并进行一对一聊天、图片传输、文件传输和本地历史记录管理。

当前项目已经从基础文本聊天原型，推进到具备完整桌面应用雏形的阶段：后端已实现局域网发现、TCP 直连消息发送、SQLite 本地持久化、媒体文件落盘、AES-GCM 本地加密、系统托盘、单实例启动和历史数据迁移；前端已实现联系人列表、聊天窗口、附件发送、图片预览、消息右键菜单、聊天搜索、多语言、多主题和存储管理等主要交互。

## 技术栈

- 桌面框架：Wails v2
- 后端语言：Go
- 前端构建：Vite
- 前端实现：原生 JavaScript、HTML、CSS
- 本地数据库：SQLite，使用 `modernc.org/sqlite`
- 局域网通信：UDP 广播发现，TCP 点对点传输
- 本地加密：AES-GCM，主密钥通过平台能力保护后保存
- 系统托盘：`github.com/getlantern/systray`

- 本地 SQLite 聊天记录持久化

- 手动添加联系人

## 已支持功能

### 1. 局域网联系人发现

应用启动后会监听固定 UDP 发现端口，并周期性广播自己的用户 ID、昵称和 TCP 监听端口。同一局域网内的其他 LanTalk 客户端收到广播后，会把对方加入联系人列表。

联系人会根据在线状态排序。自动发现的局域网联系人如果长时间未被发现，会被视为离线；如果该联系人已有聊天历史，会保留在历史联系人中，避免历史会话丢失。

### 2. TCP 点对点聊天

每个客户端启动时会创建一个本地 TCP 监听端口，用于接收其他客户端发来的消息。发送消息时，客户端会直连目标联系人的 IP 和端口。

当前支持的消息类型包括：

- 文本消息
- 图片消息
- 文件消息

文本消息限制为 2000 个字符。图片消息默认限制为 4 MB，普通文件默认限制为 20 MB。

### 3. 图片与文件消息

前端支持通过文件选择、粘贴和拖拽发送附件。图片会以图片消息展示，普通文件会以文件卡片展示。

图片消息支持：

- 在聊天窗口内预览缩略图
- 打开大图预览器
- 鼠标滚轮缩放
- 双击切换缩放倍率
- 左右方向键切换当前会话内图片
- 复制图片到系统剪贴板
- 另存为文件
- 打开所在位置

文件消息支持：

- 显示文件名、类型和大小
- 打开本地保存文件
- 另存为
- 打开所在位置
- 复制文件名
- 文件可用性检查

### 4. 消息管理

消息具备发送状态，前端会展示发送中、已发送、已接收和发送失败等状态。发送失败的出站消息可以重新发送。

单条消息支持右键操作：

- 复制文本、图片或文件名
- 另存为图片或文件
- 打开所在位置
- 重新发送失败消息
- 转发消息
- 删除消息

会话级操作包括：

- 搜索当前聊天记录
- 加载更早消息
- 清空当前会话

### 5. 本地持久化

当前版本使用 SQLite 保存核心数据，默认存储在用户配置目录下的 `LanTalk` 目录中。Windows 上通常位于：

```text
%AppData%\LanTalk
```

主要本地文件包括：

- `lantalk.db`：SQLite 数据库，保存用户信息、设置、联系人和消息记录
- `master.key`：本地加密主密钥的受保护形式
- `media/`：图片和文件等媒体内容

前端设置面板会展示当前保存目录，并支持选择新的数据目录。迁移时会复制数据库、密钥和媒体目录，成功后移除旧位置的数据文件。

### 6. 数据迁移和兼容

项目早期版本使用 `state.json` 保存状态。当前存储层保留了迁移逻辑，首次启动新版时会尝试把旧的 JSON 数据迁移到 SQLite。

存储层还包含以下兼容迁移：

- 明文 profile、peer、message 字段迁移为加密字段
- 内联图片 Data URL 迁移为独立媒体文件
- 旧图片存储路径迁移为基于哈希的去重路径
- 旧文件存储路径迁移为统一文件媒体路径

### 7. 本地加密

聊天记录中的敏感字段会在写入 SQLite 前进行加密，例如昵称、消息文本、媒体名称和媒体类型。加密使用 AES-GCM。

主密钥保存在 `master.key` 中。Windows 平台下项目提供了平台密钥保护实现，非 Windows 平台目前提供了降级实现，后续仍需要继续完善跨平台安全策略。

### 8. 媒体存储与清理

图片和文件会从消息载荷中提取出来，保存到本地 `media/` 目录。媒体内容使用 SHA-256 哈希组织路径，相同内容可以复用同一份本地文件，减少重复占用。

当前媒体存储有总量限制，默认上限为 1 GB。设置面板可以查看消息总数和媒体占用，并可以清理不再被任何聊天记录引用的孤立媒体文件。

### 9. 设置、多语言和主题

应用支持保存用户设置，包括：

- 昵称
- 语言
- 主题
- 数据保存目录

当前语言支持：

- 简体中文
- English

当前主题支持：

- 微信简约
- 暖白纸感
- 森林夜色

### 10. 桌面端体验

桌面能力包括：

- 系统托盘菜单
- 隐藏到托盘
- 从托盘恢复窗口
- 托盘显示未读消息数量
- 关闭窗口时默认隐藏到托盘
- 托盘退出应用
- 桌面通知
- 应用标题显示未读数
- 单实例锁，重复打开时唤起已运行窗口

## 开发与运行

安装前端依赖：

```bash
cd frontend
npm install
```

开发模式运行：

```bash
wails dev
```

构建桌面应用：

```bash
wails build
```

构建产物通常输出到：

```text
build/bin/LanTalk.exe
```

当前已有 Go 单元测试，主要覆盖聊天服务和媒体存储相关行为，包括：

- 历史会话联系人合并
- 离线联系人清理策略

- 旧媒体路径迁移

运行测试：

```bash

```

目前尚未看到前端自动化测试配置。前端复杂交互较多，后续建议补充关键流程的端到端测试，例如发送文本、发送附件、搜索历史、删除消息、清空会话和切换存储目录。

## 当前限制

- UDP 发现端口固定为 `48555`，同一台机器上同时运行多个真实实例并不适合作为主要测试方式。
- 局域网发现依赖广播能力，某些网络、虚拟网卡、防火墙或隔离 VLAN 环境可能影响发现结果。
- 文件和图片通过 TCP 直接发送 JSON 载荷，适合当前限制内的小型附件，尚不是高性能大文件传输协议。
- 当前未实现端到端身份认证，手动添加联系人时需要用户自行确认地址和端口可信。
- 非 Windows 平台的密钥保护能力还需要进一步强化。

## 下一步建议

建议优先推进以下工作：

1. 在两台以上真实局域网设备上做系统测试，记录发现、发送、托盘、通知和文件保存的行为。
2. 为 TCP 消息协议增加版本号和更明确的错误响应，为后续兼容升级留出空间。
3. 拆分文件传输协议，避免大附件长期走单个 JSON/Data URL 载荷。
4. 增加联系人信任或配对机制，减少误连和伪造广播风险。
5. 补充前端端到端测试，覆盖最核心的用户流程。
6. 整理发布流程，包括图标、安装器、版本号、变更记录和升级策略。
